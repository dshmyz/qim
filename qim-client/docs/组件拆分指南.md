# 组件拆分指南

## 概述

本文档说明了组件拆分的原则和实践，帮助将大型组件拆分为职责单一的组件。

## 拆分原则

### 1. 单一职责原则（SRP）

每个组件应该只有一个职责。当一个组件承担多个职责时，应该将其拆分为更小的组件。

### 2. 高内聚低耦合

- **高内聚**：相关的逻辑和状态应该放在一起
- **低耦合**：组件之间通过 props 和 events 通信，减少直接依赖

### 3. 可复用性

拆分的组件应该可以在多个场景中复用。

## 实践示例

### 示例 1: 聊天工具栏拆分

#### 原始代码（MessageInput.vue 的一部分）

```vue
<div class="input-toolbar">
  <button class="toolbar-btn" @click="$emit('start-voice-call')">
    <i class="fas fa-phone-alt"></i>
  </button>
  <button class="toolbar-btn" @click="$emit('start-video-call')">
    <i class="fas fa-video"></i>
  </button>
  <button class="toolbar-btn" @click="$emit('start-screen-share')">
    <i class="fas fa-desktop"></i>
  </button>
  <!-- 更多按钮... -->
</div>
```

#### 拆分后

1. **ChatToolbarButton.vue** - 单个按钮组件

```vue
<template>
  <button
    class="toolbar-btn"
    :class="buttonClass"
    :title="title"
    @click="$emit('click')"
    :disabled="disabled"
  >
    <i :class="icon"></i>
  </button>
</template>

<script setup lang="ts">
interface Props {
  icon: string
  title?: string
  disabled?: boolean
  variant?: 'default' | 'primary' | 'ai'
}
</script>
```

2. **ChatToolbar.vue** - 工具栏容器

```vue
<template>
  <div class="input-toolbar">
    <ChatToolbarButton
      icon="fas fa-phone-alt"
      title="语音通话"
      @click="$emit('start-voice-call')"
    />
    <ChatToolbarButton
      icon="fas fa-video"
      title="视频通话"
      @click="$emit('start-video-call')"
    />
    <!-- 更多按钮... -->
  </div>
</template>
```

## 组件通信模式

### 1. Props Down, Events Up

父组件通过 props 向下传递数据，子组件通过 emit 向上传递事件。

```vue
<!-- Parent.vue -->
<ChildComponent
  :message="message"
  @update="handleUpdate"
/>
```

### 2. Provide / Inject

对于深层嵌套的组件，使用 provide/inject 避免 prop drilling。

```ts
// Parent.vue
import { provide } from 'vue'
provide('theme', 'dark')

// Child.vue
import { inject } from 'vue'
const theme = inject('theme')
```

### 3. Pinia Store

对于跨组件共享的状态，使用 Pinia Store。

```ts
// stores/user.ts
export const useUserStore = defineStore('user', () => {
  const user = ref<User | null>(null)
  return { user }
})

// AnyComponent.vue
const userStore = useUserStore()
```

## 最佳实践

### 1. 组件命名

- 使用 PascalCase 命名（如 `ChatWindow.vue`）
- 使用简洁、描述性的名称
- 遵循项目统一的命名约定

### 2. Props 定义

- 使用 TypeScript interface 定义 props 类型
- 提供合理的默认值
- 添加必要的注释

```ts
interface Props {
  message: Message
  isOwn: boolean
  showAvatar?: boolean
}
```

### 3. Emit 定义

- 使用明确的 emit 名称
- 传递必要的参数

```ts
const emit = defineEmits<{
  'message-click': [message: Message]
  'message-delete': [messageId: string]
}>()
```

### 4. 样式管理

- 使用 scoped 样式避免污染
- 使用 CSS 变量实现主题切换
- 避免使用 !important

## 性能优化

### 1. 懒加载组件

```ts
const AsyncComponent = defineAsyncComponent(() =>
  import('./HeavyComponent.vue')
)
```

### 2. v-memo 优化

对于大量重复渲染的列表：

```vue
<div v-memo="[item.id, item.content]">
  <!-- 内容 -->
</div>
```

### 3. KeepAlive 缓存

对于需要保持状态的组件：

```vue
<KeepAlive include="ChatWindow">
  <component :is="currentView" />
</KeepAlive>
```

## 总结

组件拆分是提高代码质量和可维护性的重要手段。通过遵循单一职责原则、高内聚低耦合原则，可以创建出更加清晰、可复用和易于测试的组件。
