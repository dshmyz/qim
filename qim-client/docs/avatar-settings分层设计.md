# Avatar 设置面板分层设计

> 创建时间：2026-05-12
> 状态：待确认

## 背景

当前 Avatar 设置面板采用单层 Tab 结构，包含 7 个 Tab 项。随着功能不断增加，Tab 数量过多，新用户使用门槛较高。

## 目标

将 7 个 Tab 重新组织为两层结构：
- **第一层**：基础设置 / 高级设置（大分类）
- **第二层**：具体的功能 Tab（子分类）

## 分组方案

### 基础设置（面向普通用户）

| 子 Tab | 内容 | 说明 |
|--------|------|------|
| 基础设置 | 启用开关、分身名称、模型配置 | 核心功能，必须先配置 |
| 人设风格 | 风格学习、补充提示词、风格预览 | 定义分身的说话风格 |
| 回复策略 | 回复长度、延迟、置信度、标记样式 | 控制分身的回复行为 |

### 高级设置（面向进阶用户）

| 子 Tab | 内容 | 说明 |
|--------|------|------|
| 触发规则 | 触发模式、关键词、冷却期 | 精确控制分身何时回复 |
| 知识范围 | 会话历史、知识库、笔记、任务 | 控制分身可访问的数据范围 |
| 记忆管理 | 查看、删除、搜索记忆 | 管理分身记住的信息 |
| 技能市场 | 技能浏览、安装、管理 | 为分身添加扩展功能 |

## UI 设计

### 第一层 Tab（分类 Tab）

```
┌─────────────────────────────────────────────────────┐
│  [⚙️ 基础设置]           [🔧 高级设置]              │
└─────────────────────────────────────────────────────┘
```

样式特点：
- 高度：48px
- 字体大小：14px，font-weight: 500
- 激活状态：底部 2px 边框 + 高亮背景
- 图标 + 文字居中

样式特点：

## 状态管理

### 新增的响应式状态

```typescript
const activeCategory = ref('basic')          // 'basic' | 'advanced'
const activeBasicTab = ref('basic')          // 'basic' | 'persona' | 'reply'
const activeAdvancedTab = ref('trigger')     // 'trigger' | 'knowledge' | 'memory' | 'skills'
```

### 切换逻辑

1. 切换分类时，自动切换到该分类的默认子 Tab
2. 子 Tab 切换不影响分类状态
3. 保持每个分类的 Tab 状态独立（切回时恢复上次选择的 Tab）

## 组件复用

现有的子组件保持不变：

| 组件 | 用途 |
|------|------|
| AvatarBasicSettings | 基础设置（启用、名称、模型） |
| AvatarPersonaSettings | 人设风格（学习、补充词、预览） |
| AvatarReplySettings | 回复策略（长度、延迟、阈值） |
| AvatarTriggerSettings | 触发规则（模式、关键词、冷却） |
| AvatarKnowledgeSettings | 知识范围（历史、知识库、笔记、任务） |
| AvatarMemoryPanel | 记忆管理（查看、删除、搜索） |
| AvatarSkillsPanel | 技能市场（浏览、安装、管理） |

## 优势

1. **降低认知负担**：新用户只需关注基础设置
2. **清晰的层次**：从简单到复杂，渐进式配置
3. **更好的扩展性**：未来添加新功能时可以合理归类
4. **保持向后兼容**：子组件不需要修改

## 风险

1. **状态管理复杂度**：需要维护两层 Tab 状态
2. **默认 Tab 选择**：需要合理设置默认值

## 后续优化建议
