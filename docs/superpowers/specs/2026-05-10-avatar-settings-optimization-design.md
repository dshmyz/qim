# 分身设置界面优化设计文档

**日期**: 2026-05-10  
**作者**: AI Assistant  
**状态**: 待审查

## 1. 背景

### 1.1 问题陈述

当前分身设置界面存在以下问题：

1. **设置项过多**：7个标签页（基础设置、人设风格、触发规则、知识范围、回复策略、记忆管理、技能市场）对普通用户来说过于复杂
2. **学习成本高**：新用户需要花费较长时间理解每个设置的作用
3. **功能定位模糊**：技能市场中的工具定位不清晰，存在功能重复
4. **用户群体混合**：同时面向普通用户和高级用户，但未做区分

### 1.2 目标用户

- **普通用户**（70%）：希望快速启用分身，使用默认配置即可
- **高级用户**（30%）：希望精细控制分身行为，需要调整各种参数

### 1.3 设计目标

1. 降低普通用户的学习成本和配置负担
2. 保留高级用户的精细控制能力
3. 简化界面，提升用户体验
4. 明确功能定位，避免混淆

### 4.5 技能市场（暂时隐藏）

**决策**：暂时隐藏技能市场标签，原因如下：

1. **AI 能力工具**（知识检索、记忆检索等）是分身的基础能力，应该根据"知识来源"配置自动启用，不需要用户手动配置
2. **运维工具**（服务器监控、日志分析）是管理员工具，对普通用户没有意义
3. **功能重复**：工具开关与知识来源配置存在重复

**未来规划**：
- 当有真正的第三方工具时，重新设计技能市场
- 例如：天气查询工具、日程管理工具、外部API集成工具

### 5.2 数据流设计

```typescript
// AvatarSettingsPanel.vue
const activeTab = ref<'basic' | 'advanced'>('basic')

// 保存用户选择
watch(activeTab, (newTab) => {
  localStorage.setItem('avatar-settings-tab', newTab)
})

// 恢复用户选择
onMounted(() => {
  const savedTab = localStorage.getItem('avatar-settings-tab')
  if (savedTab === 'basic' || savedTab === 'advanced') {
    activeTab.value = savedTab
  }
})
```

### 6.2 风险评估

**潜在风险**：
1. **数据同步问题**：普通设置和高级设置共享数据，需要确保同步正确
   - 解决方案：使用 Vue 的响应式系统，确保数据一致性

2. **用户习惯改变**：老用户可能不适应新的界面
   - 解决方案：保留所有功能，只是重新组织，用户仍可访问所有设置

3. **组件复用冲突**：现有组件可能需要调整才能在新结构中使用
   - 解决方案：通过 props 和 emit 确保组件独立性
