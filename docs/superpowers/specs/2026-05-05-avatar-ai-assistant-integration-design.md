**日期**: 2026-05-05  
**状态**: 待审批

---

## 一、需求概述

### 1.1 背景

当前系统中 AI分身（Avatar）和 AI助手（AI Assistant）是两个独立的功能模块：

| 模块 | 定位 | 核心能力 |
|------|------|----------|
| **AI分身** | 用户的数字替身 | 自动学习用户风格、代理回复消息 |
| **AI助手** | 群组/个人 AI 工具 | 总结、翻译、搜索、工具调用 |

用户反馈：两个功能入口分散，选择成本高。

### 1.2 目标

将 AI分身 定位为"个人 AI 能力层"，实现以下融合目标：

1. **UI 统一**：在 AI助手 中心增加"我的分身"入口
2. **能力复用**：分身可以调用 AI助手 的工具能力
3. **体验一致**：用户在同一界面管理所有 AI 能力

## 二、架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    AI助手中心（统一入口）                      │
├────────────┬─────────────┬─────────────┬───────────────────┤
│  对话中心   │  我的机器人  │  创建机器人  │     我的分身       │
└────────────┴─────────────┴─────────────┴───────────────────┘
         │              │                │           │
         └──────────────┴────────────────┴─────┬─────┘
                                               ▼
                         ┌─────────────────────────────┐
                         │         AI 能力层（共享）     │
                         │  总结 | 翻译 | 搜索 | 工具调用 │
                         └─────────────────────────────┘
                                               │
                                               ▼
                         ┌─────────────────────────────┐
                         │       用户层（身份继承）      │
                         │     用户身份 + 会话权限       │
                         └─────────────────────────────┘
```

### 2.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **身份继承** | 分身使用主账号身份回复，带"AI代回复"标记 |
| **能力复用** | 分身可调用 AI助手 的工具能力 |
| **渐进增强** | 从 UI 整合开始，逐步增强能力 |
| **向后兼容** | 不影响现有功能的使用 |

### 3.2 Phase 2：能力复用

**工具调用场景**：

| 场景 | 工具 | 说明 |
|------|------|------|
| 智能总结回复 | 消息搜索 + 总结 | 自动查找历史上下文并总结 |
| 跨会话信息整合 | 消息搜索 | 整合其他会话的相关信息 |
| 多语言支持 | 翻译 | 自动识别并翻译外文消息 |
| 知识增强回复 | 知识库检索 | 基于文档知识生成回复 |

**API 层设计**：

```typescript
// src/api/avatar.ts
export interface AvatarWithTools {
  id: string
  enabled: boolean
  persona: AvatarPersona
  availableTools: Tool[]
  lastActiveAt: Date
}

export async function getAvatarWithTools(avatarId: string): Promise<AvatarWithTools> {
  const [avatar, tools] = await Promise.all([
    fetchAvatar(avatarId),
    fetchAITools()  // 复用 AI助手 的工具配置
  ])
  return { ...avatar, availableTools: tools }
}
```

### 3.3 Phase 3：风格学习深化

**学习来源扩展**：

| 来源 | 权重 | 说明 |
|------|------|------|
| 用户历史消息 | 60% | 主要学习来源 |
| 用户创建的 Bot 配置 | 20% | 继承用户偏好设置 |
| 用户使用 AI助手 的记录 | 20% | 学习用户的交互习惯 |

**风格应用场景**：
- 分身回复时自动应用用户风格
- 用户创建新 Bot 时推荐风格模板

## 七、安全考虑

| 风险 | 缓解措施 |
|------|----------|
| 隐私泄露 | 学习阶段过滤敏感内容；回复前敏感信息检测 |
| 权限越界 | 分身只能使用用户自己的权限；关键操作需用户确认 |
| 滥用刷屏 | 全局限流 + Worker Pool；私聊回复天然限流 |
| 内容失控 | 用户发消息即接管；消息可撤回；紧急关闭开关 |
