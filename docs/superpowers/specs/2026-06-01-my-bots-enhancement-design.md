# "我的机器人"功能增强设计方案

## 1. 功能定位

### 1.1 定位概述

**核心定位**：用户专属的场景化专家助手

**差异化价值**：在现有的"分身(Avatar)"和"群聊助手"之外，为用户提供一个**可自定义、场景化的AI助手创建平台**。

### 1.2 功能矩阵对比

| 功能 | 核心定位 | 特点 | 使用场景 |
|------|---------|------|---------|
| **分身(Avatar)** | 用户的AI化身 | 代表用户、延续对话、个性化形象 | 个性化交互、代理回复 |
| **群聊助手** | 公共智能服务 | 群内统一响应、管理助手、官方服务 | 群聊管理、公共问答 |
| **我的机器人** | 用户专属专家 | 场景专业化、自定义配置、灵活使用 | 专业领域辅助、团队协作 |

### 1.3 核心价值主张

1. **场景专业化**：用户可以创建专注于特定领域的专家机器人
2. **个性化定制**：完全掌控机器人的配置（名称、描述、系统提示词）
3. **灵活使用**：支持私聊和群聊两种模式

### 3.2 核心数据模型

```go
type Bot struct {
    ID              uint           // 主键ID
    Name            string         // 机器人名称
    Avatar          string         // 头像URL
    Description     string         // 描述
    Type            string         // 类型: system, custom, ai
    Config          string         // JSON配置（包含system_prompt等）
    IsActive        bool           // 是否启用
    CreatorID       uint           // 创建者ID
    CreatorName     string         // 创建者名称
    VirtualUserID   *uint          // 关联虚拟用户ID
    GroupID         *uint          // 群聊关联（可选）
    IsTemplate      bool           // 是否为模板
    UserConfigID    *uint          // 用户自定义模型配置ID
    UseSystemConfig bool           // 是否使用系统配置
}
```

### 3.3 创建流程优化（已实现）

**流程：**
1. 用户选择创建方式（模板 / 自定义）
2. 选择模板 → 预填充表单 → 用户编辑 → 创建
3. 直接自定义 → 使用默认提示词 → 用户编辑 → 创建

**关键代码：**
```typescript
// 模板预填充逻辑
function selectTemplate(tpl: any) {
  form.value.name = tpl.name
  form.value.description = tpl.description
  
  // 从模板配置中解析系统提示词
  let systemPrompt = ''
  if (tpl.config) {
    try {
      const config = typeof tpl.config === 'string' ? JSON.parse(tpl.config) : tpl.config
      systemPrompt = config.system_prompt || ''
    } catch (e) {
      console.error('解析模板配置失败', e)
    }
  }
  form.value.system_prompt = systemPrompt || '你是一个友好的AI助手...'
  step.value = 'custom'
}
```

### 3.4 机器人管理功能设计

**编辑功能：**
- 允许用户编辑待审批或已拒绝状态的机器人
- 编辑内容：名称、描述、系统提示词、模型配置

**删除功能：**
- 允许用户删除自己创建的机器人
- 删除前确认提示

### 3.5 群聊邀请功能设计

**流程：**
1. 用户在机器人详情页点击"邀请到群聊"
2. 选择目标群聊
3. 后端创建群聊机器人关联
4. 机器人加入群聊

## 5. 用户体验优化

### 5.1 创建流程优化

| 优化点 | 说明 |
|-------|------|
| 模板预览 | 在模板列表页展示系统提示词预览 |

| 操作按钮 | 根据状态显示可用操作 |
| 搜索筛选 | 支持按名称搜索机器人 |

## 6. 安全与权限

### 6.1 权限控制

| 角色 | 权限 |
|------|------|
| 普通用户 | 创建、编辑、删除自己的机器人 |
| 管理员 | 审批机器人、管理所有机器人 |

### 6.2 审批机制

- 用户创建机器人后，状态为"待审批"
- 管理员审批通过后，机器人才可使用
- 用户可在审批通过前编辑或删除机器人

## 8. 总结

本方案旨在将"我的机器人"定位为**用户专属的场景化专家助手**，通过以下核心改进提升价值：

1. **优化创建流程**：模板预填充 + 自定义编辑，提升用户体验
2. **完善管理功能**：支持编辑、删除机器人
3. **扩展使用场景**：支持群聊邀请
4. **清晰的定位**：与分身、群聊助手形成差异化

该方案既保留了现有功能的价值，又为未来扩展预留了空间。
