# 便签功能优化设计文档

## 背景

便签功能目前在应用列表中作为小程序存在，已支持基本的 CRUD、标签、提醒、AI 分析等功能。同时，聊天消息右键菜单已支持"保存到便签"功能。

本次优化旨在增强便签的快速捕捉能力，提供更多便捷的入口方式。

## 设计目标

1. **增强快速捕捉能力** - 提供更多便捷的便签创建入口
2. **保持界面简洁** - 不破坏现有界面布局
3. **可配置性** - 用户可根据习惯选择使用方式

## 功能设计

### P1：选中文本浮动工具栏

#### 功能描述
当用户在聊天消息中选中文本时，自动显示浮动工具栏，提供"保存到便签"快捷操作。

#### 交互流程
1. 用户在聊天消息中用鼠标选中文本
2. 选中文本后，浮动工具栏自动出现在选中文本附近
3. 浮动工具栏包含"复制"和"保存到便签"两个按钮
4. 点击"保存到便签"后：
   - 创建新便签，标题为"来自聊天"
   - 内容为选中的文本
   - 自动添加标签"聊天记录"
   - 显示成功提示

#### 技术实现要点
1. **文本选择监听**
   - 在 MessageItem 组件中监听 `mouseup` 事件
   - 使用 `window.getSelection()` 获取选中文本
   - 判断选中文本是否在消息气泡内

2. **浮动工具栏定位**
   - 使用 `Range.getBoundingClientRect()` 获取选中文本位置
   - 工具栏定位在选中文本上方居中
   - 处理边界情况（工具栏超出屏幕）

3. **便签创建**
   - 复用现有的 `addToNote` 事件机制
   - 调用便签 API 创建新便签
   - 自动添加标签和来源信息

### P2：右下角浮动按钮

#### 功能描述
在界面右下角显示一个浮动按钮，点击后弹出快速输入框，用户可以快速创建便签。

#### 交互流程
1. 浮动按钮始终显示在右下角（可配置显示/隐藏）
2. 鼠标悬停时显示提示文字"快速记录"
3. 点击按钮后弹出快速输入框
4. 输入框包含：
   - 标题输入框（可选）
   - 内容输入框（必填）
   - "保存"和"取消"按钮
5. 保存后自动关闭输入框，显示成功提示

#### 配置选项
在"设置"中添加：
- 显示/隐藏浮动按钮开关
- 浮动按钮位置选择（右下角/左下角/右侧中间）

#### 技术实现要点
1. **浮动按钮组件**
   - 创建独立的 `FloatingStickyButton.vue` 组件
   - 使用 `position: fixed` 定位
   - 实现拖拽功能（可选）

2. **快速输入框**
   - 创建 `QuickNoteInput.vue` 组件
   - 使用模态框或弹出框形式
   - 支持快捷键操作（Enter 保存，Esc 取消）

3. **配置持久化**
   - 使用 localStorage 存储用户配置
   - 在设置页面添加配置项

## API 设计

### 已有 API
- `GET /api/v1/notes` - 获取便签列表
- `POST /api/v1/notes` - 创建便签
- `PUT /api/v1/notes/:id` - 更新便签
- `DELETE /api/v1/notes/:id` - 删除便签

### 新增字段支持
创建便签时支持新增字段：
```json
{
  "title": "来自聊天",
  "content": "选中的文本内容",
  "tags": ["聊天记录"],
  "source": "chat-selection",
  "sourceData": {
    "conversationId": "xxx",
    "messageId": "yyy",
    "senderName": "张三"
  }
}
```

## 风险和缓解措施

### 风险 1：浮动按钮遮挡内容
- **缓解措施**：提供配置选项，用户可隐藏按钮或调整位置
- **备选方案**：实现拖拽功能，用户可自由调整位置

### 风险 2：文本选择体验不佳
- **缓解措施**：添加防抖，避免频繁触发
- **备选方案**：提供开关，用户可禁用此功能

### 风险 3：与现有功能冲突
- **缓解措施**：充分测试，确保不影响现有功能
- **备选方案**：提供降级方案，可快速回滚
