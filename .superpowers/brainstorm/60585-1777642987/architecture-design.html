二、数据模型（TypeScript 接口）

核心类型定义
interface Task {
id: string
title: string
description: string
status: TaskStatus // todo | in_progress | completed
priority: TaskPriority // low | medium | high
dueDate: string | null
tags: Tag[]
assignee: User | null
creator: User
subTasks: SubTask[]
commentCount: number
position: number // 拖拽排序用
createdAt: string
updatedAt: string
}
interface SubTask {
id: string
title: string
completed: boolean
position: number
}
interface Tag {
id: string
name: string
color: string // hex color
}
type TaskStatus = 'todo' | 'in_progress' | 'completed'
type TaskPriority = 'low' | 'medium' | 'high'
type TaskView = 'kanban' | 'list' | 'calendar' | 'workspace'

三、状态管理与 API 层

📦 Pinia Store: useTaskStore
State:
tasks: Task[]
currentView: TaskView
filters: TaskFilters
selectedTaskId: string | null
loading: boolean
Actions:
fetchTasks() / createTask() / updateTask()
deleteTask() / updateStatus()
reorderTask(id, position)
Getters:
filteredTasks / todoTasks
inProgressTasks / completedTasks
tasksByDate / myTasks
🔌 API 层: src/api/task.ts
GET
/api/v1/tasks → 获取任务列表
/api/v1/tasks/:id → 获取任务详情
POST
/api/v1/tasks → 创建任务
PUT
/api/v1/tasks/:id → 更新任务
/api/v1/tasks/:id/reorder → 拖拽排序
PATCH
/api/v1/tasks/:id/status → 更新状态
DELETE
/api/v1/tasks/:id → 删除任务
