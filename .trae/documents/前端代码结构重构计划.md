# QIM 前端代码重构计划

## 一、问题诊断

### 1.1 核心问题

| 问题 | 严重程度 | 详情 |
|------|----------|------|
| Main.vue 过于庞大 | 🔴 严重 | 13104 行，347KB，承担了大量不应属于它的职责 |
| 组件职责混乱 | 🔴 严重 | Sidebar.vue 已存在但 Main.vue 中仍内联实现相同功能 |
| 组件被重复定义 | 🟡 中等 | Sidebar.vue 和 Main.vue 中的侧边栏逻辑重复 |
| ChatWindow.vue 过大 | 🟡 中等 | 7611 行，包含大量业务逻辑应抽离 |
| 测试组件残留 | 🟢 轻微 | `StreamTest.vue` 未被任何地方引用 |
| Composables 缺失 | 🟡 中等 | 大量状态逻辑散布在组件中，未抽离为 composables |
| 样式组织混乱 | 🟢 轻微 | CSS 文件散落，部分 inline style，缺乏统一规范 |

### 1.2 Main.vue 问题分析

Main.vue (13104行) 实际承担了以下多个独立职责：

1. **窗口控制** — 最小化/最大化/关闭按钮（~50行）
2. **侧边栏导航** — SideOptions 图标栏（~50行），与 Sidebar.vue 功能重叠
3. **搜索功能** — 搜索结果悬浮窗（~100行）
4. **会话列表** — 最近联系人列表（~200行），应抽为 ConversationList 组件
5. **组织架构** — 组织树展示（~200行）
6. **群聊管理** — 群列表、群详情、右键菜单（~500行）
7. **应用面板** — 10+ 个 App 组件的动态加载和切换（~300行）
8. **聊天窗口** — 内联的聊天界面（~500行），但同时又引入了 ChatWindow 组件
9. **群聊创建弹窗** — ~200行
10. **用户资料弹窗** — ~100行
11. **通知中心** — ~100行
12. **WebSocket 管理** — 连接/重连/消息处理（~800行）
13. **各种右键菜单** — 群聊/联系人/消息等上下文菜单（~500行）
14. **分享功能** — ShareModal 调用和逻辑（~100行）
15. **皮肤/设置菜单** — ~200行
16. **工具函数** — request/getToken/各种格式化等（~200行）

### 1.3 组件重复问题

| 功能 | 在 Main.vue 中 | 独立组件 | 状态 |
|------|---------------|---------|------|
| 侧边栏布局 | ✅ 内联实现（第98-600行区域） | Sidebar.vue (180行) | 重复，Main.vue 未使用 Sidebar |
| 会话列表 | ✅ 内联实现 | 无独立组件 | 需要抽离 |
| 搜索结果 | ✅ 内联实现 | 无独立组件 | 需要抽离 |
| 群聊列表 | ✅ 内联实现 | GroupList.vue | 重复 |
| 组织架构 | ✅ 内联实现 | OrgTree.vue | 部分重复 |

### 1.4 ChatWindow.vue 问题分析

ChatWindow.vue (7611行) 承担的职责：

1. 消息输入和发送（~300行）
2. 消息列表展示（~200行）
3. 文件上传和预览（~400行）
4. 屏幕共享（~500行）
5. 群公告编辑（~200行）
6. 群信息编辑（~200行）
7. 成员管理（~300行）
8. 消息搜索（~200行）
9. 右键菜单（消息/成员）（~400行）
10. MiniApp 管理（~300行）
11. 各种工具函数（request/formatDate 等）（~100行）
12. CSS 样式导入（5个 CSS 文件）

## 二、重构目标

1. **Main.vue** 缩减至 500 行以内，仅负责顶层状态管理和布局编排
2. **ChatWindow.vue** 缩减至 1000 行以内
3. **消除重复组件** — 统一使用 Sidebar.vue 等独立组件
4. **移除废弃代码** — 删除未引用的 StreamTest.vue
5. **建立 composables 层** — 抽离状态逻辑
6. **规范样式组织** — 统一 CSS 模块管理

## 三、重构计划

### Phase 1: 基础建设 — 创建缺失组件和 Composables

**目标**: 为后续重构打好基础，不改变现有行为

#### 1.1 创建 ConversationList 组件

```
src/components/conversation/ConversationList.vue
src/components/conversation/SearchResult.vue
```

从 Main.vue 中提取：
- 会话列表项渲染逻辑
- 会话项右键菜单
- 搜索结果悬浮窗

#### 1.2 创建 MainLayout 组件

```
src/components/layout/MainLayout.vue
```

从 Main.vue 中提取：
- SideOptions 图标栏 → 已有 SideOptions.vue，需确认是否使用
- Window controls 窗口控制栏
- 整体布局框架

#### 1.3 创建核心 Composables

```
src/composables/useWebSocket.ts          — WebSocket 连接/消息处理
src/composables/useConversation.ts       — 会话列表/选择/搜索

src/composables/useScreenShare.ts        — 屏幕共享逻辑
src/composables/useContextMenu.ts        — 右键菜单通用逻辑
```

### Phase 2: Main.vue 瘦身

**目标**: 将 Main.vue 从 13104 行缩减至 500 行以内

#### 2.1 替换内联侧边栏为 Sidebar 组件

- Main.vue 当前在第 98-700 行左右内联实现了侧边栏
- 应改为使用 `Sidebar.vue` 组件，通过 props/events 通信
- 需要确保 Sidebar.vue 能覆盖当前内联实现的所有功能

#### 2.2 替换内联会话列表为 ConversationList 组件

- 将 Main.vue 中的会话列表渲染逻辑移入 ConversationList.vue
- 将搜索结果渲染逻辑移入 SearchResult.vue

#### 2.3 抽离 WebSocket 逻辑到 composable

- 将 Main.vue 中的 WebSocket 连接/重连/消息处理移至 `useWebSocket.ts`
- 预计可移除 ~800 行代码

#### 2.4 抽离群聊管理逻辑

- 将群聊相关操作（创建、解散、右键菜单）移至 `useGroup.ts` composable
- 预计可移除 ~500 行代码

#### 2.5 清理模板中的重复渲染逻辑

- Main.vue template 中大量条件渲染可简化
- 使用组件替代内联模板

### Phase 3: ChatWindow.vue 瘦身

**目标**: 将 ChatWindow.vue 从 7611 行缩减至 1000 行以内

## 五、风险点和注意事项

1. **行为一致性** — 重构期间必须保证功能不变，每个阶段完成后需要手动验证
2. **Main.vue 和 Sidebar.vue 的冲突** — 需要仔细确认 Sidebar.vue 是否完整覆盖 Main.vue 中的侧边栏功能
3. **WebSocket 状态管理** — 抽离 WebSocket 逻辑时需确保消息处理不丢失
4. **渐进式重构** — 每完成一个阶段就提交，避免大跨度改动导致难以定位问题
5. **测试策略** — 当前项目无自动化测试，需要人工在每个阶段后验证核心功能
