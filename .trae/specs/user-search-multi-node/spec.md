# QIM 产品需求文档

## 概述
- **摘要**：实现最近会话的用户搜索功能和服务端多节点部署能力，提升用户体验和系统可靠性。
- **目的**：解决用户快速找到联系人并发起聊天的需求，同时提高系统的可扩展性和可用性。
- **目标用户**：QIM 应用的所有用户和系统管理员。

## 目标
- 实现最近会话的用户搜索功能，支持按照中文名和账号搜索
- 优化搜索结果的展示和交互，点击搜索结果可直接发起聊天
- 实现服务端多节点部署，提高系统的可用性和扩展性
- 确保多节点部署时的数据一致性和负载均衡

## 非目标（范围外）
- 不修改现有聊天功能的核心逻辑
- 不涉及数据库架构的重大变更
- 不实现跨数据中心的部署方案

## 背景与上下文
- QIM 是一款企业级即时通讯应用，目前已实现基本的聊天功能
- 当前系统采用单节点部署，随着用户量增长，需要支持多节点部署
- 用户搜索功能目前仅支持会话名称搜索，需要扩展到用户搜索

## 功能需求
- **FR-1**：最近会话搜索功能支持按中文名搜索用户
- **FR-2**：最近会话搜索功能支持按账号搜索用户
- **FR-3**：搜索结果展示用户列表，点击可直接发起聊天
- **FR-4**：服务端支持多节点部署
- **FR-5**：多节点部署时保持数据一致性
- **FR-6**：多节点部署时实现负载均衡

## 非功能需求
- **NFR-1**：搜索响应时间不超过 500ms
- **NFR-2**：服务端节点故障时自动切换到其他节点
- **NFR-3**：系统支持水平扩展，可动态添加节点
- **NFR-4**：多节点部署时消息传递延迟不超过 100ms

## 约束
- **技术**：前端使用 Vue 3，后端使用 Go
- **依赖**：需要消息队列或类似机制保证多节点间的数据同步
- **业务**：保持现有功能的完整性，不影响已有用户的使用

## 假设
- 前端已有的搜索框组件可以复用
- 后端需要添加用户搜索的 API 接口
- 服务端多节点部署需要使用负载均衡器

## 验收标准

### AC-1：用户搜索功能
- **给定**：用户在最近会话页面的搜索框输入中文姓名
- **当**：用户输入搜索关键词
- **然后**：搜索结果显示匹配的用户列表
- **验证**：`programmatic`
- **备注**：搜索结果应包含用户头像、姓名和账号

### AC-2：账号搜索功能
- **给定**：用户在最近会话页面的搜索框输入账号
- **当**：用户输入搜索关键词
- **然后**：搜索结果显示匹配的用户列表
- **验证**：`programmatic`

### AC-3：点击搜索结果发起聊天
- **给定**：搜索结果显示用户列表
- **当**：用户点击某个搜索结果
- **然后**：系统创建与该用户的私聊会话并进入聊天界面
- **验证**：`programmatic`

### AC-4：服务端多节点部署
- **给定**：系统部署在多个服务器节点上
- **当**：客户端发起请求
- **然后**：请求被负载均衡器分配到不同节点
- **验证**：`programmatic`

### AC-5：多节点数据一致性
- **给定**：系统部署在多个节点上
- **当**：一个节点收到消息
- **然后**：其他节点也能获取到该消息
- **验证**：`programmatic`

### AC-6：节点故障自动切换
- **给定**：系统部署在多个节点上，其中一个节点故障
- **当**：客户端发起请求
- **然后**：请求被自动分配到正常节点
- **验证**：`human-judgment`
