# 笔记模块 AI 增强设计

## 概述

对 QIM 笔记模块进行 AI 增强优化，包括界面布局改进、AI 分析功能、标签系统和导出功能。

## 目标

- 优化笔记编辑器布局，采用单栏切换模式
- 增加 AI 分析功能，自动生成摘要、推荐标签、提取行动项
- 增强标签系统，支持筛选和管理
- 支持导出 Markdown 文件
- 优化笔记卡片样式

## 非目标

- 不改动笔记核心数据结构（仅增加字段）
- 不重构笔记编辑器核心逻辑
- 不实现与聊天的关联功能（已有便签承担此职责）

## 功能需求

### FR-1：编辑器布局优化

- 采用单栏切换模式，默认显示编辑区
- 顶部增加切换按钮：`[编辑] [预览]`
- 预览模式下只读，显示渲染后的 Markdown
- 切换时保持滚动位置

### FR-2：工具栏优化

顶部工具栏，包含以下按钮：
- 保存：保存笔记
- AI 分析：分析笔记，生成摘要、推荐标签、提取行动项
- 导出：导出为 Markdown 文件
- 分享：分享笔记（复用现有功能）
- 删除：删除笔记（二次确认）

### FR-3：笔记卡片优化

- 标题更突出（字号 16px，加粗）
- 摘要区显示 AI 生成的摘要（如果有），否则显示内容前 50 字
- 标签使用药丸样式，可点击筛选
- hover 时显示操作按钮

### FR-4：标签系统

- 笔记列表顶部增加标签筛选器
- 点击标签筛选相关笔记
- AI 分析时自动推荐标签
- 编辑笔记时可手动添加/删除标签

### FR-5：AI 分析功能

点击"AI 分析"按钮后：
- 自动生成摘要（显示在笔记列表预览中）
- 推荐标签（用户可确认或修改）
- 提取行动项（可选：一键转为任务）

**AI 服务复用**：
- 复用项目已有的 `ai.AIService`（位于 `qim-server/ai/ai_service.go`）
- 使用用户配置的 AI 模型（支持 OpenAI、阿里通义、腾讯混元等）

**分析 Prompt 模板**：
```
你是一个笔记分析助手。分析以下笔记内容，返回 JSON 格式结果：
1. summary: 笔记摘要（100字以内）
2. tags: 推荐标签（最多5个）
3. action_items: 提取的行动项（如果有）

笔记内容：
{note_content}

返回格式：
{"summary": "...", "tags": ["标签1", "标签2"], "action_items": ["行动项1", "行动项2"]}
```

### FR-6：导出 Markdown

- 点击"导出"按钮
- 下载 `.md` 文件，文件名为笔记标题
- 包含完整的 Markdown 内容

## 数据模型改动

Note 模型增加字段：

```typescript
interface Note {
  id: number
  title: string
  content: string
  type: 'note' | 'sticky'
  tags: string[]          // 新增：标签数组
  summary: string         // 新增：AI 生成的摘要
  created_at: string
  updated_at: string
}
```

## 后端 API 改动

| API | 方法 | 说明 |
|-----|------|------|
| `/api/v1/notes/:id/analyze` | POST | AI 分析笔记，返回摘要、标签、行动项 |
| `/api/v1/notes/:id/export` | GET | 导出笔记为 Markdown 文件 |
| `/api/v1/notes/:id` | PUT | 更新笔记（包含 tags、summary） |

## 验收标准

### AC-1：编辑器切换

- **给定**：用户打开一个笔记
- **当**：用户点击"预览"按钮
- **则**：编辑区切换为预览区，显示渲染后的 Markdown
- **验证**：`programmatic` - 切换按钮状态正确，内容正确渲染

### AC-2：AI 分析

- **给定**：用户编辑了一篇笔记
- **当**：用户点击"AI 分析"按钮
- **则**：显示 AI 分析结果弹窗，包含摘要、推荐标签、行动项
- **验证**：`human-judgment` - 分析结果与笔记内容相关

### AC-3：标签筛选

- **给定**：笔记列表中有多篇带标签的笔记
- **当**：用户点击某个标签
- **则**：列表只显示包含该标签的笔记
- **验证**：`programmatic` - 筛选结果正确

### AC-4：导出 Markdown

- **给定**：用户打开一个笔记
- **当**：用户点击"导出"按钮
- **则**：下载 `.md` 文件，内容与笔记一致
- **验证**：`programmatic` - 文件内容正确

## 改动量评估

| 改动项 | 前端 | 后端 | 复杂度 |
|--------|------|------|--------|
| 单栏编辑器 | 改造编辑器布局 | 无 | 低 |
| 工具栏优化 | 调整按钮位置和样式 | 无 | 低 |
| 卡片样式优化 | CSS 调整 | 无 | 低 |
| 标签系统 | 列表筛选 + 标签编辑 | Note 模型增加字段 | 中 |
| AI 分析 | 分析按钮 + 结果弹窗 | 新增 API + AI 调用 | 中 |
| 导出 Markdown | 导出按钮 | 新增 API | 低 |

**总体评估**：增量式改动，不涉及核心逻辑重构，预计 1-2 周完成。
